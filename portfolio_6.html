end-to-end batch pipeline
EDA — distribution + importance analysis
SalePrice right-skewed → log transform. LightGBM normalised feature importance (%) to identify where to spend feature engineering effort. Correlation heatmap to find strong predictors. Removed features with <1% importance
Null handling — structural absence flags
Nulls where structure is absent (no garage, no basement) → created has_flag binary feature to preserve signal. Remaining nulls filled at neighbourhood level average rather than global average
Skew correction + redundancy removal
Right-skewed → log transform. Left-skewed → reflect (MAX+1 - value) then log transform. Correlation > 0.8 between features → dropped one (e.g. GarageArea vs GarageCars removed)
Feature creation + outlier removal
New features: total bathroom area, combined 1st+2nd floor area, total porch area, basement total + finished. Outliers removed via Area vs SalePrice scatter inspection
Leakage-safe ColumnTransformer pipeline
Ordinal encoding for quality columns. One-hot for low cardinality categoricals. Target encoding for high cardinality categoricals. StandardScaler for numerical. PCA for linear regression models. All transformations inside single ColumnTransformer — fit on train only to prevent data leakage
Model selection via cross-validation
Trained Linear Regression, Ridge, Lasso, XGBoost and LightGBM. Compared CV mean RMSE and CV std dev across splits — low std = model stable across folds. LightGBM selected as best performer
RMSE 0.13 · ~14% error · log-transformed target
key insight: nulls caused by structural absence (no garage = null GarageArea) carry information — a has_flag captures this signal. All encoding and scaling wrapped inside ColumnTransformer makes the pipeline leakage-safe — transformations fitted on train data only, applied consistently to test. Target encoding for high-cardinality columns prevents one-hot explosion while preserving the category-to-price relationship.
